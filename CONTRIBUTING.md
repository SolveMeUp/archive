# 기여 가이드

`archive`는 [SolveMeUp](https://solvemeup.com)의 자료구조·알고리즘 지식 아카이브입니다.
누구나 글을 제안할 수 있고, 큐레이터의 리뷰를 거쳐 반영됩니다.

## 라이선스 동의 (중요)

이 저장소에 기여하면, 본인의 기여물이 저장소와 동일한 조건으로 제공되는 데 동의하는 것으로 간주합니다.
별도의 CLA는 없습니다.

- **글(산문)** 은 [CC BY-SA 4.0](LICENSE)으로 제공됩니다.
- **코드 예제(스니펫)** 는 자유 사용으로 제공됩니다. (저작권을 주장하지 않습니다)

## 제안 방법

직접 push 권한은 큐레이터에게만 있습니다. 그 외에는 **fork → Pull Request** 흐름을 따릅니다.

1. 이 저장소를 fork 합니다.
2. 브랜치를 만들어 작업합니다. (예: `add/dijkstra`)
3. 글을 작성/수정한 뒤 PR을 엽니다.
4. 큐레이터 1명 이상의 승인을 받으면 `main`에 squash 머지됩니다.

## 글 작성 규칙

### 위치와 구조

- 모든 글은 `content/` 아래 카테고리 폴더에 둡니다.
- **한 글 = 폴더 하나**이며, 폴더 안에 다음을 둡니다.
  - `index.md` — 본문 (필수)
  - 이미지 등 에셋 — 같은 폴더에 함께 (선택)
- 폴더 경로(슬러그)는 소문자-하이픈(kebab-case)으로 짓습니다.

```
content/graph/shortest-path/dijkstra/
├── index.md
└── example.png        (그림이 있다면)
```

### 카테고리

카테고리(분류 체계)는 관리자가 관리합니다. 아래 분류에 맞는 위치에 글을 추가해 주세요.
(해당 폴더가 아직 없으면 글과 함께 만들면 됩니다.)

```
content/
├── data-structure/
├── graph/
│   ├── tree/
│   ├── shortest-path/
│   ├── mst/
│   ├── scc/
│   └── network-flow/
├── dp/
│   └── optimization/
├── string/
├── math/
│   ├── number-theory/
│   ├── combinatorics/
│   ├── linear-algebra/
│   └── polynomial/
├── geometry/
├── technique/
├── greedy/
└── game-theory/
```

- **새 카테고리가 필요하면** PR에 새 폴더를 임의로 만들지 말고, 먼저 **Issue로 제안**해 주세요.
  큐레이터 합의 후 위 목록에 추가됩니다.

### 내용

- **형식은 자유입니다.** 정해진 글 템플릿을 강제하지 않습니다 —
  알고리즘마다 자연스러운 설명 흐름이 다르기 때문입니다.
- 개념 설명과 **수도코드**를 기본으로 권장합니다.
- 언어별 구현(Java/Python/C++ 등)은 **환영하지만 필수는 아닙니다.** 한 언어만 있어도 됩니다.
- 수식은 KaTeX 문법(`$...$`, `$$...$$`)을 사용합니다.
- 코드 블록에는 언어를 명시합니다. (예: ` ```cpp `)
- **자기 말로 서술**해 주세요. 번역·복붙이 아닌 독창적 설명을 원칙으로 하며,
  참고한 자료가 있으면 출처를 남깁니다.

> 글 본문에 메타데이터(frontmatter)는 아직 두지 않습니다. 순수 마크다운으로 작성해 주세요.

## 스타일

- 한 PR은 하나의 주제에 집중합니다. (글 1개 또는 명확한 묶음 단위)
- 깨진 링크·잘못된 마크다운 문법이 없는지 확인하고 올립니다.

질문이나 제안은 Issue로 남겨 주세요.
