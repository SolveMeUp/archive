# archive

[SolveMeUp](https://solvemeup.com)의 **자료구조·알고리즘 지식 아카이브**입니다.
알고리즘 레퍼런스를 모아, 그 자체로 독립적인 지식 자산이면서
SolveMeUp 본 서비스 안에서 읽을 수 있는 것을 목표로 합니다.

## 어떻게 굴러가나요

- **콘텐츠**는 이 저장소의 마크다운 파일로 관리됩니다. (public 저장소)
- **제안·승인**은 GitHub의 fork → PR → 머지 흐름으로 이루어집니다.
- **표현**(렌더링)은 이 저장소가 **정적 사이트로 직접 빌드**하여 담당합니다 —
  마크다운을 정적 HTML로 구워(KaTeX 수식 + 코드 하이라이팅 포함) `solvemeup.com/archive`에서 서빙합니다.

아카이브는 SolveMeUp에 종속되지 않습니다. 의존 방향은 **SolveMeUp → archive** 한쪽뿐이라,
"이 글에 맞는 문제 추천" 같은 연동은 서비스 쪽이 담당하고(연결 키는 글의 `id`), 아카이브 글은 알고리즘 설명에만 집중합니다.
별도의 백엔드 서버나 데이터베이스는 두지 않습니다 — 정적 빌드 산출물을 서빙할 뿐입니다.

## 구조

모든 콘텐츠는 `content/` 아래 카테고리 폴더에 들어갑니다. 최상위 카테고리는 다음과 같습니다.

`data-structure` · `graph` · `dp` · `string` · `math` · `geometry` · `technique` · `greedy` · `game-theory`

각 카테고리의 하위 분류를 포함한 전체 트리는 [CONTRIBUTING.md의 카테고리](CONTRIBUTING.md#카테고리)에서 확인할 수 있습니다.

- **한 글 = 폴더 하나**입니다. 폴더 안에 본문 `index.md`(필수)와 이미지 등 에셋(선택)을 함께 둡니다.
  - 예: `content/graph/shortest-path/dijkstra/index.md`
- 폴더 경로가 곧 그 글의 식별자(슬러그)입니다.
- 카테고리(폴더 구조)는 관리자가 관리합니다. 새 카테고리가 필요하면 Issue로 먼저 제안해 주세요.

## 글의 성격

- **언어:** 한국어로 작성합니다. (코드·알고리즘명은 영어)
- **범위:** 기본부터 심화까지 다룹니다.
- **내용:** 개념 설명과 수도코드를 기본으로 하고, 언어별 구현(Java/Python/C++ 등)은 환영합니다(필수 아님).
- **서술:** 번역·복붙이 아닌 자기 말로 서술합니다. 참고 자료는 출처를 표시합니다.

## 기여하기

누구나 글을 제안할 수 있습니다. [CONTRIBUTING.md](CONTRIBUTING.md)를 참고해 주세요.
모든 변경은 큐레이터의 리뷰를 거쳐 `main`에 반영됩니다.
